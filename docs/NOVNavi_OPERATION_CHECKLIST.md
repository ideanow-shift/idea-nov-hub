# NOV Navi 動作確認チェックリスト

更新後やHUB側へ渡す前に、上から順番に確認します。

## 1. ログイン

- [ ] `honbu / nov-admin` でログインできる
- [ ] 本部アカウントだけ右上に `管理` が表示される
- [ ] 通常店舗では `管理` が表示されない

期待結果: Supabase `concierge_store_credentials` で認証される。スプレッドシート/GASは通常経路では使わない。

## 2. スタッフ質問

- [ ] クイックチップから質問できる
- [ ] 入力欄から自由質問できる
- [ ] 回答文が表示される
- [ ] 関連リンクが表示される
- [ ] 回答評価ボタンが表示される
- [ ] 回答ルールに当たらない質問は本部確認の注意表示が出る

期待結果: スタッフはNotebookLMやスプレッドシートを意識せず、NOV Navi内で回答を受け取れる。

## 3. ログ保存

- [ ] 質問後、Supabase `concierge_question_logs` に行が追加される
- [ ] 店舗ID、質問、回答、Notebookカテゴリ、取得元、リスク、要確認が保存される
- [ ] 回答評価を押すとSupabase `concierge_feedback` に保存される

期待結果: `source = fallback` の質問は管理画面で `未整備候補` として確認できる。

## 4. 本部管理画面

- [ ] 質問ランキングが表示される
- [ ] 店舗別利用数が表示される
- [ ] 検索ワードランキングが表示される
- [ ] 要確認・低評価質問が表示される
- [ ] 質問ログを評価、取得元、日付で絞り込める
- [ ] CSV出力に `取得元`、`リスク`、`要確認` が含まれる

期待結果: 本部が問い合わせ傾向と未整備候補を見て、回答ルールやナレッジを改善できる。

## 5. 回答ルール管理

- [ ] 既存の回答ルール一覧が表示される
- [ ] 新しい回答ルールを追加できる
- [ ] 関連リンクをチェックボックスで選択できる
- [ ] リスクと本部確認を設定できる
- [ ] `リスク = 高` の場合、本部確認が必須になる
- [ ] リスクフィルターで `通常`、`注意`、`高`、`本部確認` を絞り込める
- [ ] 追加したキーワードで質問すると、新しい回答が表示される

期待結果: 本部スタッフがスプレッドシートを直接触らず、管理画面から回答を増やせる。

## 6. ナレッジ管理

- [ ] `正本資料を開く` から該当Google Driveフォルダを開ける

- [ ] Supabase `concierge_knowledge_updates` に保存される

期待結果: 正本資料、NotebookLM、更新履歴がHUB管理画面からつながる。

## 7. リンク管理

- [ ] 登録済みリンク一覧が表示される
- [ ] URL、表示名、カテゴリ、担当、説明、状態を更新できる

期待結果: フォームURL変更時も、コードやスプレッドシートを触らず管理画面から直せる。
